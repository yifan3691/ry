# 实现 Excel 上传动态建表功能计划

我们将实现一个“上传即建表”的功能：用户上传 Excel，系统根据文件名作为表名，第一行作为列名自动在数据库中创建表并导入数据。如果表已存在，则直接导入数据（或跳过）。

## 1. 后端实现 (模块: `ods-business`)

我们将复用项目中已有的 `ods-business` 模块来实现业务逻辑，避免污染核心框架。

### 1.1 数据层 (Mapper)

创建 `OdsUploadMapper`，负责执行动态 SQL。

* **`checkTableExists(tableName)`**: 查询 `information_schema.tables` 判断表是否存在。

* **`createTable(sql)`**: 执行动态生成的 `CREATE TABLE` 语句。

* **`insertData(params)`**: 执行动态生成的 `INSERT INTO` 语句。

### 1.2 业务层 (Service)

创建 `OdsUploadService`，核心逻辑如下：

1. **解析 Excel**: 使用 Apache POI (`WorkbookFactory`) 读取上传的文件。
2. **确定表名**:

   * 规则：`ods_` + 文件名（去除扩展名）。

   * 处理：将中文转为拼音或直接使用（MySQL支持中文表名，建议转义），过滤特殊字符防止 SQL 注入。
3. **读取表头**: 获取第一行作为列名列表。
4. **动态建表**:

   * 如果表不存在：生成 SQL `CREATE TABLE table_name (id BIGINT PRIMARY KEY AUTO_INCREMENT, col1 VARCHAR(500), col2 VARCHAR(500)...)` 并执行。

   * 如果表存在：对比 Excel 列名与数据库现有列名是否一致（简单起见，仅校验列数量或直接尝试插入）。
5. **数据导入**:

   * 遍历剩余行，构建批量插入 SQL。

   * 执行数据入库。

### 1.3 控制层 (Controller)

创建 `OdsUploadController`。

* 接口：`POST /ods/upload`

* 参数：`MultipartFile file`

* 逻辑：调用 Service 并返回结果（如“创建表成功，导入 100 条数据”）。

## 2. 前端实现 (Vue)

### 2.1 新增页面

创建 `src/views/ods/upload/index.vue`。

* 使用 `el-upload` 组件实现文件上传。

* 添加“拖拽上传”区域，提升体验。

* 上传成功后显示后端返回的处理结果（如“表 ods\_user 已存在，新增数据 5 条”）。

### 2.2 菜单配置

* 在系统管理 -> 菜单管理中新增“Excel 动态导入”菜单，指向上述组件路径。

## 3. 技术细节与约束

* **依赖**: 使用项目现有的 `org.apache.poi:poi-ooxml`，无需引入新依赖。

* **列类型**: 默认所有自动创建的列类型为 `VARCHAR(500)` 或 `TEXT`，以容纳各种格式数据，避免类型转换错误。

* **安全性**: 对表名和列名进行正则校验，防止 SQL 注入。
